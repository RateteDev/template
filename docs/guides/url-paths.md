# URL パス設計ガイド

## 基本方針

URL はユーザーが読み、共有し、記憶するインターフェースとして設計する。
短く、予測可能で、意味が読み取れる構造にする。

## 多言語パス構造

多言語サイトの URL 構造は [i18n ガイド](./i18n.md) に従う。

## 静的アセットの公開パス

### アイコン

アイコンの配置と公開パスは [アイコンガイド](./icons.md) に従う。

公開 URL: `/assets/icons/{icon-name}.svg`

### 画像・その他のアセット

```
/assets/images/*    — 写真・イラスト等
/assets/fonts/*     — フォントファイル
/assets/icons/*     — アイコン（アイコンガイド参照）
```

アセットの種別ごとにディレクトリを分離する。種別の異なるファイルを混在させない。

## 命名規則

| 対象 | 規則 | 例 |
|------|------|----|
| パスセグメント | kebab-case | `/ja/getting-started` |
| ファイル名 | kebab-case | `arrow-right.svg` |
| クエリパラメータ | camelCase または snake_case（プロジェクトで統一） | `?pageSize=10` |

- 大文字は使用しない（URL は case-sensitive だが、ユーザーが入力・記憶しやすいよう小文字に統一する）
- 末尾スラッシュのルールはフレームワークの規約に従い、プロジェクト内で統一する
