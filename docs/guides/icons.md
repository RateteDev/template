# アイコンガイド

## 基本方針

アイコンの独自作成は極力避け、既存のオープンソースアイコンセットを使用する。
統一感・トレーサビリティ・ライセンス管理の観点から、以下のルールに従う。

## アイコンセットの選定

Tabler Icons を優先して使用する。

- 4900 超の収録数で網羅性が高い
- ストロークベースで線幅が統一されており、視覚的な一貫性を保ちやすい
- MIT ライセンスで商用利用可

### Tabler Icons にないアイコンが必要な場合

安易に別ソースから追加せず、以下の順に検討する。

1. Tabler Icons の既存アイコンで代替できないか再検討する
2. プロジェクトごとにフォールバック基準（ライセンスがプロジェクトの要件と一致すること等）を設け、適合する場合のみ別ソースから採用する
3. 上記でも代替できない場合のみ、自作する

2・3 で追加するアイコンには以下を適用する。

- 線幅・サイズ・スタイルを Tabler Icons に合わせて調整する
- 本ガイドのトレーサビリティ・ライセンスルールをすべて適用する

## ファイル管理

### ディレクトリ構造（物理配置）

アイコンは静的ファイルとして管理する。アイコンコンポーネントライブラリ（`@tabler/icons-react` など、動的インポートや tree shaking に依存するパッケージ）は使用しない。ローカルに配置した SVG をバンドラーでビルド時に取り込むことは差し支えない。

物理配置はフレームワークの静的ファイル配信規約に従い、画像・写真と混在させず専用ディレクトリに分離する。

```
{static-root}/assets/icons/*.svg
```

`{static-root}` はフレームワークにより異なる（`public/`, `static/` など）。

### 配信方式

公開 URL を持つ静的配信か、ビルド時インライン展開かは、アイコンごとの消費経路で判断する。

- JS 実行を介さず外部から直接フェッチされる用途（favicon、外部埋め込みなど）がある場合は、公開 URL を持つ静的配信とする
- それ以外は、UI 側の要件（`currentColor` 継承による動的配色の要否など）に応じてどちらを選んでもよい

いずれの方式でも、SVG は source コメントの付与を除き改変せず、上記ディレクトリに物理配置する。自作アイコンも同様に SVG として配置し、TS/JS へのパスデータの手動書き写しは行わない。

### 公開 URL

公開 URL を持たせる場合、フレームワークの配信規約に関わらず以下に統一する。

```
/assets/icons/{icon-name}.svg
```

### パスエイリアス

アイコンへの参照は、フレームワークのパスエイリアス機能で短縮することを推奨する。エイリアス名や設定方法はフレームワークの作法に従う。

```
@icons/{icon-name}.svg  →  {static-root}/assets/icons/{icon-name}.svg
```

## トレーサビリティ

### SVG 冒頭コメント

ファイル単体で出自を追跡できるよう、すべての SVG ファイルの先頭に取得元を記載する。
取得元の種別（Tabler Icons・他ライブラリ・自身の別リポジトリや Web サイトなど）を問わず、フォーマットは同一。

```xml
<!-- source: {source-name}/{icon-name} | url: {download-url} -->
<svg ...>
```

例: `<!-- source: tabler-icons/home | url: https://tabler.io/icons/icon/home -->`

取得元となる外部の管理主体が存在しない（このプロジェクトのために新規作成した）アイコンのみ、以下とする。
自作理由や設計判断の経緯はコミットメッセージまたは docs に記載し、本コメントには含めない。

```xml
<!-- source: original | url: none -->
<svg ...>
```
