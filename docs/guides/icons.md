# アイコンガイド

## 基本方針

アイコンの独自作成は極力避け、既存のオープンソースアイコンセットを使用する。
統一感・トレーサビリティ・ライセンス管理の観点から、以下のルールに従う。

## アイコンセットの選定

Tabler Icons を優先して使用する。

- 4900 超のアイコンを収録しており、網羅性が高い
- ストロークベースで線幅が統一されており、視覚的な一貫性を保ちやすい
- MIT ライセンスで商用利用可

### Tabler Icons にないアイコンが必要な場合

安易に別ソースから追加しない。以下の順に検討する。

1. Tabler Icons の既存アイコンで代替できないか再検討する
2. プロジェクトごとにフォールバック基準を設け、基準に適合する場合のみ別ソースから採用する
   - MIT ライセンスであること
   - 線幅・サイズ・スタイルを Tabler Icons に合わせて調整すること
   - 同一ガイドのトレーサビリティ・ライセンスルールをすべて適用すること

## ファイル管理

### ディレクトリ構造（物理配置）

アイコンは静的ファイルとして管理する。フレームワークの動的インポートや tree shaking には依存しない。

物理的なディレクトリ配置はフレームワークの静的ファイル配信規約に従う。
アイコンは画像・写真と混在させず、専用ディレクトリに分離する。

```
{static-root}/assets/icons/*.svg
```

`{static-root}` はフレームワークにより異なる（`public/`, `static/` など）。

### 公開 URL

フレームワークの配信規約に関わらず、公開 URL は以下に統一する。

```
/assets/icons/{icon-name}.svg
```

### パスエイリアス

アイコンへの参照は、フレームワークのパスエイリアス機能で短縮することを推奨する。

```
@icons/{icon-name}.svg  →  {static-root}/assets/icons/{icon-name}.svg
```

エイリアス名やその設定方法はフレームワークの作法に従う。

## トレーサビリティ

### SVG 冒頭コメント

すべての SVG ファイルの先頭に、取得元を以下のフォーマットで記載する。
ファイル単体で出自を追跡できるようにするため。

```xml
<!-- source: tabler-icons/{icon-name} | {download-url} -->
<svg ...>
```

Tabler Icons 以外のソースの場合も同一フォーマットに従う。

```xml
<!-- source: {library-name}/{icon-name} | {download-url} -->
<svg ...>
```

### ライセンスファイル

プロジェクトの `/licenses` に、使用しているアイコンの法的根拠を集約する。
ライセンス全文と使用アイコンの一覧（アイコン名・取得元 URL）を含める。
具体的なページ構成やフォーマットはプロジェクトごとに定める。
