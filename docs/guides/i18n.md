# i18n ガイド

## 基本方針

すべてのサイトで多言語対応を前提に設計する。
対応言語が1つであっても、URL構造・テンプレート・コンテンツ管理を多言語に対応した形で構築する。

## URL構造

すべての言語にロケールプレフィックスを付ける。デフォルト言語も例外としない。

```
/ja/docs    → 日本語
/en/docs    → 英語
```

ロケールなしパスは言語判定リダイレクト専用とし、コンテンツを持たせない。

```
/docs       → 302 リダイレクト → /{lang}/docs
/           → 302 リダイレクト → /{lang}/
```

### 判定の優先順位

ロケールなしパスへのアクセス時に、以下の優先順位で言語を判定する。

1. **Cookie** — ユーザーが過去に言語セレクターで選択した言語
2. **Accept-Language** — ブラウザの言語設定
3. **デフォルト言語** — フォールバック（日本語）

ロケール付きパスへのアクセス時は、URL で指定された言語をそのまま使用する。
Cookie や Accept-Language による上書きは行わない。

### リダイレクトのステータスコード

ユーザーの状態（Cookie, Accept-Language）によって遷移先が変わるリダイレクトには 302 または 307 を使用する。
301 / 308 はブラウザや CDN に恒久的にキャッシュされ、状態の変更が反映されなくなるため、言語判定リダイレクトには使用しない。

| 種別 | コード | 例 |
|------|--------|-----|
| 言語判定リダイレクト | 302 / 307 | `/docs` → `/ja/docs` |
| 恒久的な URL 移転 | 301 / 308 | `/old-page` → `/new-page` |

## 判定場所

言語判定とリダイレクトは、利用可能な最も手前のレイヤーで処理する。

| 優先度 | レイヤー | 例 |
|--------|---------|-----|
| 1 | エッジ | Cloudflare Workers, Vercel Edge Middleware |
| 2 | サーバーミドルウェア | Hono `app.use()`, Express `app.use()` |
| 3 | フレームワークミドルウェア | Astro `src/middleware.ts`, Next.js `middleware.ts` |

クライアントサイド（JavaScript）での判定は、一瞬デフォルト言語が表示されるフラッシュが発生し、SEO 的にも不利なため使用しない。

## Cookie による言語設定の永続化

言語設定は Cookie に保存する。
言語設定 Cookie は Strictly Necessary Cookie（技術的に必要な Cookie）に分類され、同意バナーなしで設定できる。

### Cookie を更新するタイミング

Cookie の更新は、ユーザーが UI の言語セレクターを明示的に操作したときのみ行う。
ロケール付き URL を閲覧しただけでは Cookie を更新しない。

共有リンクや検索結果から別言語のページを一度開いただけで、以後の言語設定まで変更されることを防ぐため。

### UI の言語セレクター

すべてのページに言語セレクターを設置する。
言語セレクターの操作時に以下を行う。

1. Cookie を選択された言語に更新する
2. 現在のページの対応言語 URL に遷移する（例: `/ja/docs` → `/en/docs`）

## SEO

### hreflang

各言語ページに hreflang タグを相互に設定する。
`x-default` にはロケールなし URL を指定する。

```html
<!-- /ja/docs と /en/docs の両方に同じセットを記載 -->
<link rel="alternate" hreflang="ja" href="https://example.com/ja/docs" />
<link rel="alternate" hreflang="en" href="https://example.com/en/docs" />
<link rel="alternate" hreflang="x-default" href="https://example.com/docs" />
```

### ロケールなし URL のインデックス防止

ロケールなし URL はリダイレクト専用であり、コンテンツを持たない。
検索エンジンにインデックスされないよう、レスポンスヘッダに `X-Robots-Tag: noindex` を付与する。

## リンクの共有

サイトの共有リンクやスライドに記載する URL は、ロケールなしを推奨する。
受け手の言語設定に合わせた表示になり、URL も短くなる。

特定の言語で読んでほしい場合は、ロケール付き URL を使用する。

| 目的 | 共有する URL |
|------|-------------|
| 相手の言語に任せる（推奨） | `https://example.com/docs` |
| 特定言語で見せたい | `https://example.com/ja/docs` |
