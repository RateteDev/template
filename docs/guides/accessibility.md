# アクセシビリティガイド

## 基本方針

見た目や操作性をカスタマイズする場合も、ブラウザ標準の挙動とアクセシビリティは内部的に保持する。
装飾は理解・操作を助ける場合にのみ使う。意味のない装飾のためにマークアップの意味を犠牲にしない。

## セマンティックHTML

要素は視覚効果ではなく意味で選ぶ。`div` / `span` の濫用を避け、`button` / `a` / `ul` / `li` など意味に応じたタグを使う。

### 見出し階層

見出しは情報構造をユーザーと支援技術に伝えるために使う。見た目を大きくしたいだけで使わない。

- `h1`: ページ全体の主題のみ。コンポーネント内部やカード内で使わない。
- `h2`: セクション主見出し。`h3`: 小見出し。`h4`: さらに細かな区分が必要な場合のみ。
- **`h5` 以降は禁止。** ユーザー入力・生成AI出力でも許可しない。入力された場合は変換/保存/生成のいずれかの段階で弾くか、制約を短く説明する(黙って変換しない)。
- `h4` を超える階層化が必要なら、見出しを深くするのではなく情報設計を見直す: ページ分割、タブ分割、カード化、リスト化、ラベル、強調表現。

## ARIA・支援技術対応

- アイコンのみで意味を伝える操作要素には `aria-label` 等を付与する。
- キーボード操作・フォーカスリングを損なわない実装にする。独自コンポーネントでネイティブ要素を置き換える場合も、同等の操作性を担保する。
- フォームやファイル選択などのインタラクティブ要素をカスタムUIで包む場合も、ネイティブ要素(`input`, `file input` 等)のキーボード操作・スクリーンリーダー対応は内部的に保持する。

## 禁止事項

- 視覚的な見た目のためだけに見出しタグ・セマンティック要素を選択/省略しない。
- `h5` 以降の見出しタグを使用しない。
- キーボード操作・スクリーンリーダー対応を損なうカスタムUIへの完全置換。
